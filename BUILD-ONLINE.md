# Сборка .exe онлайн через GitHub (без установки Node.js)

Этот способ позволяет получить готовый `.exe` **не устанавливая ничего на свой компьютер**. GitHub сам соберёт приложение на виртуальной Windows-машине.

## Пошаговая инструкция (~10 минут)

### 1. Создайте аккаунт GitHub
Если ещё нет — регистрация на [github.com](https://github.com/signup) (бесплатно).

### 2. Создайте новый репозиторий
- Нажмите "+" в правом верхнем углу → **New repository**
- Имя: `apple-notes-clone` (или любое другое)
- Приватный или публичный — неважно, сборка работает в обоих
- **НЕ** ставьте галочки "Add README" / "Add .gitignore" — у нас свои
- Нажмите **Create repository**

### 3. Загрузите файлы проекта

**Самый простой способ — через веб-интерфейс:**

- На странице пустого репозитория нажмите **"uploading an existing file"** (ссылка синим текстом)
- Распакуйте ZIP-архив у себя на компьютере
- **Перетащите все файлы и папки** (включая скрытые: `.github/`, `.gitignore`) из распакованной папки `apple-notes-clone/` в окно браузера
- Внизу страницы: **Commit changes**

> ⚠️ Важно: перетащите именно **содержимое** папки `apple-notes-clone/`, а не саму папку. То есть в корне репозитория должны оказаться `package.json`, `src/`, `assets/`, `.github/` и т.д.

> ⚠️ Папка `.github` с точкой может быть скрыта в проводнике Windows. Включите отображение скрытых файлов: в проводнике **Вид → Показать → Скрытые элементы**.

### 4. Сборка запустится автоматически
Как только файлы загружены, GitHub сам запустит сборку. Проверить можно так:
- Во вкладке **Actions** (вверху страницы репозитория) вы увидите запущенный workflow "Build Windows .exe"
- Подождите ~5 минут (зелёная галочка = готово)

### 5. Скачайте готовый .exe
- Кликните на завершённую сборку во вкладке **Actions**
- Прокрутите вниз до раздела **Artifacts**
- Там будет два архива:
  - **Notes-Installer** — установщик NSIS (`.exe`, который ставит программу на ПК)
  - **Notes-Portable** — портативная версия (`.exe`, запускается без установки)
- Нажмите на название → скачается `.zip`, внутри лежит `.exe`

## Если что-то пошло не так

**Вкладка Actions пустая.** Файл `.github/workflows/build-windows.yml` не загрузился. В проводнике включите отображение скрытых файлов и перетащите папку `.github` заново.

**Сборка упала с красным крестиком.** Кликните на неё → посмотрите логи. Чаще всего — опечатка в загруженных файлах. Пришлите мне скриншот ошибки, разберём.

**Хочу пересобрать после правок.** Редактируете файлы прямо на GitHub (иконка карандаша) → Commit. Сборка запустится автоматически. Или вручную: Actions → Build Windows .exe → **Run workflow**.

## Ограничения бесплатного GitHub Actions
- **2000 минут сборки в месяц** на приватных репозиториях (одна сборка ~5 мин = хватит на 400 сборок)
- **Неограниченно** на публичных репозиториях
- Артефакты хранятся 30 дней, потом удаляются (можно скачать и сохранить у себя)
